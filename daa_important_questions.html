Group B & C — Subjective Questions

1. Define minimum spanning tree. Differentiate between Prim's Algorithm and Kruskal's Algorithm.

Minimum Spanning Tree (MST): Given a connected, undirected, and weighted graph, an MST is a spanning tree whose sum of edge weights is minimized.

Differences:

  • Approach: Prim's builds the tree vertex by vertex from a starting node. Kruskal's builds it edge by edge, picking the smallest edge that doesn't form a cycle.
  • Data Structures: Prim's uses a Priority Queue (Min-Heap). Kruskal's uses a Disjoint-Set (Union-Find) and sorting.
  • Graph Type: Prim's is generally faster for dense graphs, while Kruskal's is better for sparse graphs.
  • Connectivity: In Prim's, the growing MST is always connected. In Kruskal's, it may be a disconnected forest until the very end.

2. What is Dynamic Programming? How does it differ from the Greedy Method?

Dynamic Programming (DP): An algorithmic paradigm that solves a given complex problem by breaking it into subproblems and stores the results of subproblems to avoid computing the same results again. It applies when the problem has optimal substructure and overlapping subproblems.

Differences from Greedy Method:

  • Choice: Greedy makes a locally optimal choice at each step without looking ahead. DP considers all possible choices recursively (or bottom-up) to find the global optimum.
  • Optimality: Greedy does not always guarantee an optimal solution. DP always guarantees an optimal solution if the problem has optimal substructure.
  • Efficiency: Greedy is generally faster and uses less memory. DP can be slower and requires memory for memoization/tabulation.
  • Direction: Greedy is top-down. DP is usually bottom-up.

3. What is the time complexity of fun(int n)?

int fun(int n) {
    int count = 0;
    for (int i = n; i > 0; i /= 2) {
        for (int j = 0; j < i; j++) {
            count += 1;
        }
    }
    return count;
}

Analysis: The outer loop halves i at each step: n, n/2, n/4, ..., 1. The inner loop runs i times.

Total iterations of the inner loop = n + n/2 + n/4 + ... + 1.

This is a geometric series sum: n(1 - (1/2)^log n) / (1 - 1/2) ≈ 2n.

Time Complexity: O(n)

4. Differentiate between Big-O and Omega notation with an example.

  • Big-O (O) Notation: Represents the upper bound of an algorithm's time complexity. Formally, f(n) = O(g(n)) if there exist constants c > 0 and n₀ > 0 such that 0 ≤ f(n) ≤ c * g(n) for all n ≥ n₀.
  • Omega (Ω) Notation: Represents the lower bound of an algorithm's time complexity. Formally, f(n) = Ω(g(n)) if there exist constants c > 0 and n₀ > 0 such that 0 ≤ c * g(n) ≤ f(n) for all n ≥ n₀.

Example: Insertion Sort.

In the worst case (reverse sorted array), time taken is quadratic. So, Worst Case = O(n²).
In the best case (already sorted array), time taken is linear. So, Best Case = Ω(n).

5. What kind of problems can we solve using dynamic programming? Can we solve the all-pair shortest path problem using Dijkstra algorithm?

Problems suitable for DP: Problems that exhibit two key properties:

  1. Overlapping Subproblems: The problem can be broken down into subproblems which are reused several times.
  2. Optimal Substructure: The optimal solution to the problem incorporates the optimal solutions to the subproblems.

Examples include Knapsack, Matrix Chain Multiplication, and Longest Common Subsequence.

Can we solve All-Pairs Shortest Path using Dijkstra?

Yes, we can run Dijkstra's algorithm V times, treating each vertex as the source once. For a graph with non-negative edge weights, the time complexity would be O(V * (V + E) log V). However, Dijkstra's cannot handle negative weight edges, unlike the Floyd-Warshall algorithm (which is a DP approach designed for the all-pairs problem).

7. Write the Ford-Fulkerson algorithm for maximum flow in a network.

Ford_Fulkerson(Graph G, Node S, Node T):
  Initialize flow f = 0 for all edges
  Initialize residual graph G_f to be the same as G
  
  While there exists an augmenting path P from S to T in G_f:
      Find the minimum capacity c_min on the path P
      Add c_min to the total flow: f = f + c_min
      
      For each edge (u, v) in P:
          Decrease capacity of (u, v) in G_f by c_min
          Increase capacity of reverse edge (v, u) in G_f by c_min
          
  Return total flow f

The augmenting path is typically found using BFS (which makes it the Edmonds-Karp algorithm) or DFS.

10. Write the difference between dynamic programming and divide and conquer algorithm. Define greedy algorithm.

Dynamic Programming vs. Divide & Conquer:

  • Subproblems: Divide and Conquer breaks a problem into independent subproblems (e.g., Merge Sort). DP breaks a problem into overlapping subproblems.
  • Memoization: Divide and Conquer solves each subproblem from scratch. DP stores the results of subproblems to avoid redundant work.
  • Use Case: Divide and Conquer is generally for problems that don't have overlapping subproblems. DP is for optimization problems with overlapping subproblems and optimal substructure.

Greedy Algorithm Definition:

An algorithm that makes the locally optimal choice at each stage with the hope of finding a global optimum. It never reconsiders its choices. It is generally faster but doesn't guarantee an optimal solution for all problems.

11. Explain the concept of backtracking and how it is related to recursion. Write the difference between backtracking and branch and bound.

Backtracking: An algorithmic technique that builds a solution incrementally, piece by piece. If at any point it determines that the current partial solution cannot lead to a valid complete solution, it abandons it ("backtracks") and tries another path. It is fundamentally related to recursion because it uses a recursive call stack to implicitly explore a state space tree.

Backtracking vs. Branch and Bound:

  • Exploration: Backtracking typically uses Depth-First Search (DFS). Branch and Bound typically uses Breadth-First Search (BFS) or Best-First Search.
  • Purpose: Backtracking is used for finding all valid solutions or any valid solution (e.g., N-Queens, Sudoku). Branch and Bound is exclusively used for optimization problems (finding the best solution, e.g., TSP).
  • Pruning: Backtracking prunes when a constraint is violated. Branch and bound prunes when a path's bound (estimated cost) is worse than the best solution found so far.

12. Define the single source shortest path problem. When does the shortest path not exist?

Single Source Shortest Path Problem: Given a graph G=(V, E) and a source vertex 'S', find the shortest paths from 'S' to all other vertices in V. The path length is the sum of the edge weights along the path.

When does it not exist?

The shortest path does not exist (is undefined) if the graph contains a negative weight cycle reachable from the source. In such a cycle, one could loop infinitely to achieve an arbitrarily low (negative infinity) cost, making the concept of a "shortest" path meaningless.

13. Define the characteristics of the greedy algorithm. Why we cannot solve 0/1 knapsack algorithm using greedy algorithm?

Characteristics of Greedy Algorithms:

  • They make locally optimal choices at each step.
  • They do not reconsider past choices (no backtracking).
  • They are usually very efficient (often linear or O(n log n) if sorting is required).

Why it fails for 0/1 Knapsack:

In 0/1 Knapsack, we must take an item entirely or leave it. The greedy approach (e.g., taking items with highest value/weight ratio first) might leave empty space in the knapsack because an item is too large to fit. A dynamic programming approach is needed to explore combinations of items that maximize profit while perfectly utilizing the available capacity.

15. Write the Floyd-Warshall Algorithm for all-pairs shortest path problem. What will be the time complexity?

Floyd_Warshall(W[V][V]):
  Let D = W  // Initialize distance matrix with edge weights
  For k = 1 to V:
      For i = 1 to V:
          For j = 1 to V:
              D[i][j] = min(D[i][j], D[i][k] + D[k][j])
  Return D

Time Complexity: The algorithm has three nested loops, each iterating V times.

Time Complexity: O(V³)

17. Write short note on Randomized algorithms.

Randomized Algorithms: An algorithm that employs a degree of randomness as part of its logic. It uses random bits to guide its behavior, in hopes of achieving good performance in the "average case" over all possible choices of random numbers.

Two main types:

  1. Las Vegas Algorithms: Always produce the correct result, but their runtime is a random variable (e.g., Randomized Quick Sort).
  2. Monte Carlo Algorithms: Have a deterministic runtime, but their output has a small probability of being incorrect (e.g., Karger's Min-Cut algorithm, Miller-Rabin primality test).

18. Define the concept of an algorithm and explain its characteristics.

Algorithm: A finite sequence of well-defined, unambiguous instructions to solve a specific problem or perform a computation.

Characteristics:

  • Finiteness: Must terminate after a finite number of steps.
  • Definiteness: Each step must be precisely defined and unambiguous.
  • Input: Has zero or more inputs supplied externally.
  • Output: Produces at least one output.
  • Effectiveness: Operations must be basic enough that they can, in principle, be done exactly and in a finite length of time by someone with paper and pencil.

20. Write the algorithm for single source shortest path (-ve edge weights allowed) [Bellman-Ford].

Bellman_Ford(Graph G, Node S):
  Initialize distance to all vertices as Infinity
  Initialize distance to S as 0
  
  For i = 1 to |V| - 1:
      For each edge (u, v) with weight w in G:
          If dist[u] + w < dist[v]:
              dist[v] = dist[u] + w
              
  // Detect negative weight cycle
  For each edge (u, v) with weight w in G:
      If dist[u] + w < dist[v]:
          Return "Graph contains negative weight cycle"
          
  Return dist

21. Discuss the advantages and drawbacks of Dynamic Programming.

Advantages:

  • Optimality: Guarantees finding the globally optimal solution.
  • Efficiency over Naive Recursion: Reduces time complexity drastically (often from exponential to polynomial) by avoiding redundant calculations via memoization/tabulation.

Drawbacks:

  • Space Complexity: Can require significant memory to store the DP table (tabulation), which can be O(n²) or worse.
  • Applicability: Only works if the problem has optimal substructure and overlapping subproblems.
  • Complexity: Formulating the recursive state and transitions can be difficult to design compared to greedy approaches.

23. Write and explain the classes P, NP, NP-Hard and NP complete.

  • P (Polynomial Time): The set of decision problems solvable by a deterministic Turing machine in polynomial time.
  • NP (Non-deterministic Polynomial Time): The set of decision problems where a "yes" answer can be verified in polynomial time by a deterministic Turing machine. P ⊆ NP.
  • NP-Hard: A problem is NP-Hard if every problem in NP can be reduced to it in polynomial time. They are at least as hard as the hardest problems in NP.
  • NP-Complete: A problem is NP-Complete if it is both in NP AND is NP-Hard. If any NP-Complete problem can be solved in polynomial time, then P = NP.

24. Explain: "If NP complete problem can be solved in polynomial time, then P=NP"

By definition, a problem L is NP-Complete if every problem in NP can be reduced to L in polynomial time. If L itself can be solved in polynomial time, then for any problem X in NP, we can first reduce X to L in polynomial time, and then solve L in polynomial time. The composition of two polynomial-time operations is also polynomial time. Thus, every problem X in NP would be solvable in polynomial time, meaning NP ⊆ P. Since P is already a subset of NP, this implies P = NP.

25. What is reducibility?

Reducibility is a transformation of one problem into another problem. If a problem A can be reduced to a problem B in polynomial time, it means we can use an algorithm that solves B to solve A with at most polynomial overhead.

If A is reducible to B (denoted as A ≤p B), then B is at least as hard as A. This concept is fundamentally used to prove NP-Completeness.

28. Write the algorithm to solve Fractional Knapsack problem.

Fractional_Knapsack(Weights, Profits, Capacity, n):
  Calculate ratio[i] = Profits[i] / Weights[i] for all i
  Sort items in descending order of ratio[i]
  
  total_profit = 0
  For i = 0 to n-1:
      If Weights[i] <= Capacity:
          Take item i entirely
          total_profit += Profits[i]
          Capacity -= Weights[i]
      Else:
          Take fraction of item i to fill remaining capacity
          fraction = Capacity / Weights[i]
          total_profit += Profits[i] * fraction
          Capacity = 0
          Break
          
  Return total_profit

31. Shortest path on graph with negative weight cycle: i) Can valid shortest path be determined? ii) Which algorithm can detect?

i) No. A valid shortest path cannot be determined because you can loop through the negative weight cycle infinitely to keep decreasing the path cost towards negative infinity.

ii) Bellman-Ford Algorithm can detect negative weight cycles. It does this by running the relaxation step one extra time (V-th time). If any distance is updated during this V-th iteration, it confirms the presence of a negative weight cycle.

32. Proof that Clique Decision problem is NP-Complete.

  1. Prove Clique is in NP: Given a graph G, an integer k, and a certificate (a set of k vertices), we can verify in polynomial time (O(k²)) whether every pair of vertices in the set is connected by an edge.
  2. Prove Clique is NP-Hard: We reduce a known NP-Complete problem, like 3-SAT, to Clique. For any 3-SAT formula, we can construct a graph where each node represents a literal in a clause. We draw edges between nodes in different clauses as long as they are not logical negations of each other. A satisfying assignment exists if and only if there is a k-clique in the graph (where k is the number of clauses). This reduction takes polynomial time.

Since it is in NP and NP-Hard, it is NP-Complete.

35. Write the algorithm for the matrix chain multiplication.

Matrix_Chain_Order(p):
  n = length[p] - 1
  Let m[1..n, 1..n] and s[1..n-1, 2..n] be new tables
  
  For i = 1 to n:
      m[i, i] = 0
      
  For L = 2 to n:  // L is the chain length
      For i = 1 to n - L + 1:
          j = i + L - 1
          m[i, j] = Infinity
          For k = i to j - 1:
              q = m[i, k] + m[k+1, j] + p[i-1]*p[k]*p[j]
              If q < m[i, j]:
                  m[i, j] = q
                  s[i, j] = k
                  
  Return m and s

36. Describe Circuit Satisfiability problem and prove that Circuit SAT is in NP.

Circuit Satisfiability (Circuit-SAT) Problem: Given a boolean combinatorial circuit composed of AND, OR, and NOT gates, is there an assignment of truth values to the inputs that makes the output of the circuit TRUE?

Proof that it is in NP:

  1. Certificate: The certificate is a specific assignment of truth values (0s and 1s) to the inputs of the circuit.
  2. Verification: Given this assignment, a deterministic Turing machine can evaluate the output of each gate layer by layer in topological order. Since the number of gates is finite (size of the circuit), evaluating the whole circuit takes time proportional to the number of gates, which is polynomial time.

Since a solution can be verified in polynomial time, Circuit-SAT is in NP.
